# 诗语家居后端开发计划（Java 8）

## 1. 计划说明

### 1.1 编制依据
本计划基于以下文档统一制定：
- `prd.md`（需求范围、优先级、业务规则、非功能要求）
- `frontapi.md`（前后端接口契约，覆盖首页/商品/交易/售后/会员等）
- `backDesign.md`（后端架构与技术选型，Java 8 + Spring Boot）
- `init.sql`（数据库初始化结构与索引设计）

### 1.2 开发目标
在 Java 8（JDK 1.8）规范下，按 P0→P1 逐步交付小程序电商后端，优先实现完整交易闭环：
- 浏览商品 → 规格选择 → 加购结算 → 下单支付 → 发货履约 → 售后退款
- 支撑后台商品/订单/售后/运营基础管理能力

### 1.3 范围边界
- 本计划聚焦后端服务、数据库、接口、中间件与交付节奏
- 小程序前端页面开发不在本计划实施内容内（仅接口联调依赖）

## 2. 技术与架构基线

## 2.1 固定技术栈
- 语言：Java 8（JDK 1.8）
- 框架：Spring Boot 2.7.x、Spring MVC、Spring Security + JWT
- ORM：MyBatis-Plus 3.5.x
- 数据库：MySQL 8.0（CloudBase MySQL）
- 缓存：Redis 6.x（缓存、分布式锁、幂等控制）
- 消息：RabbitMQ（支付回调、发货通知、积分发放等异步化）
- 存储：腾讯云COS / 阿里云OSS（商品图、售后凭证图）

## 2.2 架构实施策略
结合当前项目阶段，采用“两阶段架构”降低复杂度：
1. **阶段 A（MVP）**：单体模块化（推荐）
   - `user/product/order/payment/aftersale/marketing/admin` 在单仓内按模块分层
   - 优点：迭代快、联调成本低、便于快速上线
2. **阶段 B（扩展）**：按业务域拆分微服务
   - 当并发/团队规模增长后再拆分，保留服务边界设计

## 2.3 数据库落地约束（来自 `init.sql`）
- 全表保留 `_openid` 字段，用于用户隔离与权限控制
- 默认不依赖外键，采用“索引 + 应用层约束”
- 关键交易表（`orders`、`order_items`、`payment_records`、`stock_reservations`）需要强事务与幂等控制

## 5. 模块开发清单（WBS）

## 5.1 用户与认证模块
- 数据表：`users`、`user_addresses`、`user_favorites`、`user_footprints`、`search_histories`
- 关键能力：微信登录、JWT 签发、个人信息维护、地址管理
- 重点风险：手机号隐私保护、Token 续期与踢出策略

## 5.2 商品与搜索模块
- 数据表：`categories`、`products`、`product_skus`、`product_spec_*`、`filter_*`、`search_*`
- 关键能力：分类树、SKU 规格联动、筛选聚合、热词/联想词
- 重点风险：筛选性能、SKU 缺货态一致性

## 5.3 交易与库存模块
- 数据表：`cart_items`、`orders`、`order_items`、`stock_reservations`、`inventory_logs`
- 关键能力：价格试算、库存锁定、订单创建、超时关闭、幂等控制
- 重点风险：超卖、重复下单、并发取消与支付竞态

## 5.4 支付与履约模块
- 数据表：`payment_records`、`shipments`、`shipment_tracks`、`order_logs`
- 关键能力：微信支付、回调验签、发货与物流轨迹
- 重点风险：回调重复通知、支付成功后落库失败

## 5.5 售后与营销模块
- 数据表：`after_sale_*`、`coupons`、`coupon_scope_relations`、`user_coupons`、`user_points_*`
- 关键能力：售后状态机、退款协同、优惠券核销、积分账本
- 重点风险：退款与订单状态一致性、优惠叠加规则冲突

## 5.6 管理后台模块
- 数据表：`admin_users`、`admin_roles`、`admin_user_roles`、`operation_logs`
- 关键能力：RBAC 权限、操作审计、运营配置管理
- 重点风险：越权访问、审计日志缺失

## 6. 关键技术专项计划

## 6.1 一致性与幂等
- 下单幂等：`user_id + client_token` 防重
- 支付回调幂等：`out_trade_no` 唯一与状态机校验
- 库存一致性：Redis 分布式锁 + MySQL 乐观锁 + 异步补偿

## 6.2 性能优化
- 缓存优先：首页配置、分类树、热词、专题内容
- 分页策略：列表接口统一游标/偏移分页规范
- 索引治理：基于 `init.sql` 索引复核慢 SQL

## 6.3 安全与合规
- 敏感信息加密与脱敏（手机号、地址）
- 支付回调签名校验
- 接口限流、黑名单、设备指纹（防刷）

## 6.4 可观测性
- 日志：请求链路日志 + 业务审计日志
- 指标：QPS、RT、错误率、支付成功率、库存扣减失败率
- 告警：支付回调失败、订单积压、库存异常

## 8. 依赖与风险管理

## 8.1 外部依赖
- 微信登录与微信支付商户配置
- 物流轨迹第三方服务可用性
- COS/OSS 存储权限与 CDN 域名配置

## 8.2 主要风险与应对
| 风险 | 影响 | 应对方案 |
| --- | --- | --- |
| 支付回调异常/重复 | 订单状态错乱 | 回调幂等、消息补偿、定时对账 |
| 高并发超卖 | 订单履约失败 | 分布式锁 + 乐观锁 + 超卖告警 |
| 优惠叠加规则复杂 | 金额计算错误 | 规则引擎化、试算接口与回归样例 |
| 售后与退款链路复杂 | 用户投诉 | 审核状态机、操作留痕、超时任务 |
| 多模块并行开发冲突 | 进度延误 | API 优先、契约测试、周节点评审 |
