# 需求规格说明书（PRD）- 诗语家居（Yu Home）

## 1. 引言
### 1.1 项目背景
`诗语家居` 是一个面向微信生态的家纺类电商小程序，核心经营床品、被子、枕头、凉席、婚嫁套件及专柜系列商品。产品不仅要满足基础电商交易闭环，还要解决家纺行业中“规格复杂、材质差异大、场景感强、售后敏感”的决策问题。

### 1.2 项目目标
- 构建稳定、流畅、易转化的微信小程序商城。
- 完成“浏览商品 -> 选择规格 -> 下单支付 -> 发货履约 -> 售后服务”的完整闭环。
- 通过尺寸助手、材质说明、场景化展示降低用户决策成本。
- 建立优惠券、积分、储值卡等复购机制，提升复购率与客单价。

## 2. 产品定位与设计原则
### 2.1 产品定位
- 品类定位：家纺/卧室家居用品零售。
- 用户定位：追求舒适度、注重卧室氛围、重视材质与品质的家庭用户及个人消费者。
- 价值定位：精选商品、清晰规格、安心售后、场景化导购。

### 2.2 设计原则
1. 场景化展示：商品详情与首页专题突出卧室氛围、婚嫁场景、季节主题。
2. 低决策成本：把尺寸、材质、套件组成、洗护说明以结构化方式表达清楚。
3. 多规格清晰：规格切换后价格、库存、图片、默认数量要实时联动。
4. 交易透明：优惠、运费、库存、售后状态可追踪。
5. 可运营增长：支持 Banner、专题、优惠券、会员积分、老客复购。

## 3. 用户角色
- 普通用户（C 端）：浏览商品、收藏、加购、下单、支付、查看物流、申请售后。
- 运营人员（B 端）：维护 Banner、专题、优惠券、商品上下架、推荐位。
- 客服/售后人员（B 端）：处理退款、退货退款、换货、物流异常。
- 管理员（B 端）：商品、订单、用户、权限、数据看板统一管理。

## 4. 业务范围与优先级
- P0：首页、分类、搜索、商品详情、购物车、订单确认、微信支付、地址管理、订单查询、基础售后、后台商品/订单/用户管理。
- P1：尺寸助手、材质知识说明、物流轨迹、搭配推荐、积分体系、收藏/足迹、专题运营。
- P2：分销中心、会员等级、礼品卡、拼团/秒杀等深度营销玩法。

## 6. 关键业务规则
### 6.1 规格与价格规则
- 商品按 SPU + SKU 管理，SPU 负责商品主体信息，SKU 负责价格、库存、规格组合。
- 列表页展示 `min_price`；详情页根据当前选中 SKU 展示实际售价与库存。
- 不同规格可配置独立图片，避免用户误选颜色或材质。

### 6.2 库存规则
- 下单提交时校验可售库存，支付成功后扣减库存。
- 支付前可采用锁库存机制，超时未支付自动释放。
- 售罄 SKU 不允许加入购物车和立即购买。

### 6.3 订单规则
- 待付款订单超过设定时长自动关闭。
- 订单、订单明细、地址、价格、规格均需保存快照，避免商品变更影响历史订单。
- 运费、优惠、积分抵扣、储值卡抵扣需分别记账。

### 6.4 售后规则
- 每个订单明细可单独发起售后申请。
- 售后类型包括：仅退款、退货退款、换货。
- 售后需记录申请原因、图片凭证、审核结论、退款结果。

### 6.5 会员资产规则
- 积分发放与抵扣需可追溯。
- 优惠券需校验使用门槛、有效期、适用品类和是否与其他优惠叠加。
- 储值卡消费、退款、余额变更需产生独立流水。

## 7. 非功能性需求
- 性能：首屏加载时间目标小于 1.5 秒，列表分页与图片懒加载必须启用。
- 并发：满足日常 500+ 同时在线浏览，并具备活动期弹性扩容能力。
- 安全：手机号、地址等敏感信息加密存储；支付回调验签；接口具备幂等处理。
- 兼容性：兼容主流 iOS/Android 微信版本。
- 可观测性：核心链路需记录日志、埋点、错误告警，支持订单与支付问题排查。
- 合规：符合微信小程序审核规范，商品宣传、支付、隐私协议和售后条款需完整披露。
